# RS-31_Haponko_Nikita_RTOS_LR8

**Номер групи:** РС-31

**Автор:** Гапонько Нікіта

**ОПЕРАЦІЙНІ СИСТЕМИ РЕАЛЬНОГО ЧАСУ ТА 32-БІТНІ МК**
**Лабораторна робота 8.** Технологія використання черг та засобів синхронізації при розробці ПЗ в системі FreeRTOS.

**Обладнання:** STM32F411 Discovery, CubeMX, Keil.

**Мета:** Вивчити механізми міжпроцесної взаємодії (IPC) у FreeRTOS, навчитися використовувати черги (Queues) для передачі даних та синхронізації роботи декількох потоків.

__________________________________________________________________________________________

**Завдання до лабораторної роботи (Варіант 4):**
1. Створити чотири потоки для керування світлодіодами **PD12**, **PD13**, **PD14**, **PD15**.
2. Реалізувати механізм передачі «естафети»: натискання кнопки **PA0** надсилає повідомлення у чергу першого потоку (PD12). 
3. Після активації свого світлодіода, кожен потік має передати повідомлення наступному потоку в ланцюжку (PD12 -> PD13 -> PD14 -> PD15).

Цей проєкт демонструє складну координацію між незалежними завданнями. Замість прямого керування, потоки «спілкуються» між собою, що є основою архітектури надійних систем реального часу.

### 👨🏼‍💻 Функціонал:
* **Inter-task Communication**: Використання черг FreeRTOS для передачі керуючих токенів між задачами.
* **Sequential Execution**: Чітка послідовність спрацювання індикації, що ініціюється зовнішньою подією (кнопкою).
* **Queue-based Synchronization**: Потоки перебувають у заблокованому стані (не споживають ресурси процесора), поки в їх чергу не надійде повідомлення.

### 📋 Ключові моменти реалізації:
1. **xQueueCreate**: Створення декількох об'єктів черг для організації зв'язків між конкретними задачами.
2. **Non-blocking Waiting**: Використання `xQueueReceive` з нескінченним часом очікування (`portMAX_DELAY`).
3. **ISR to Task Interaction**: Коректна передача сигналу від кнопки з обробника переривання до першого потоку за допомогою `xQueueSendFromISR`.
